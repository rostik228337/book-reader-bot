# ЗАДАЧА

В этой папке сейчас лежит рабочий Telegram-бот на aiogram 3 (`bot/`), который задаёт пользователю вопросы по главам книги через инлайн-кнопки. Нужно **переделать его в Telegram Mini App** — полноценное веб-приложение внутри Telegram с кольцевым прогресс-баром и строгой логикой прохождения.

Существующий код — полезная точка отсчёта (`bot/grader.py`, `data/chapters.json`, `bot/storage.py` и т.д.), но архитектура меняется: вместо aiogram-хендлеров будет FastAPI-бэкенд + HTML/JS фронт.

## Что должно получиться

1. **Telegram-бот** — минимальный, одна кнопка «Открыть приложение» (WebApp).
2. **Mini App** — HTML+JS-фронт, открывается внутри Telegram. На главном экране:
   - Большое кольцо с процентом прочитанного (например `30%`). Процент = `(пройденных глав / всего глав) * 100`.
   - Список глав — с замочками / галочками.
3. **Экран главы** — вопросы один за другим. Пользователь вводит ответ текстом.
4. **Строгая логика прохождения (ВАЖНО):** глава засчитывается, **ТОЛЬКО если на КАЖДЫЙ вопрос получил 2 балла**. Если хоть один ответ оценён в 0 или 1 → глава НЕ пройдена, прогресс не растёт, следующая глава остаётся закрытой. Пользователь должен перечитать главу и пройти её с самого первого вопроса заново.
5. **Back-end** — FastAPI + SQLite, оценка ответов по ключевым словам с лемматизацией (pymorphy3), без ИИ.

# СТЕК

**Строго этот:**
- Python 3.11+
- FastAPI + uvicorn (веб-сервер + API)
- aiogram 3.x (минимальный бот только ради кнопки WebApp)
- aiosqlite (async SQLite)
- pymorphy3 + pymorphy3-dicts-ru (лемматизация русского)
- python-dotenv
- Ванильный HTML + CSS + JavaScript на фронте (**БЕЗ** React/Vue/TS/Vite — чтобы не ломать голову со сборкой и хостингом)
- Telegram Web App SDK — подключить `<script src="https://telegram.org/js/telegram-web-app.js">`

Не использовать: SQLAlchemy, Docker, React, TypeScript, webpack, вебхуки.

# ДЕТАЛИ

## 1. Бот (`backend/bot.py`)

Только одна функция:
- Команда `/start` → отвечает текстом «👋 Привет! Нажми кнопку, чтобы открыть приложение» + инлайн-кнопка `InlineKeyboardButton(text="📖 Открыть", web_app=WebAppInfo(url=config.webapp_url))`.
- Также установить **Menu Button** через `bot.set_chat_menu_button` при старте (тип `MenuButtonWebApp`, тот же URL) — чтобы кнопка приложения была всегда видна слева от поля ввода.

## 2. FastAPI (`backend/main.py`)

Роуты:
- `GET /` → возвращает `webapp/index.html`
- `GET /app.js`, `GET /style.css` → отдаёт статику (или через `StaticFiles`, смонтировать `webapp/` на `/`).
- `POST /api/init` — принимает `{initData: str}`, валидирует через `auth.verify_init_data(initData, BOT_TOKEN)`. Возвращает `{user: {id, first_name}, books: [...], progress: {...}, total_progress_percent: float}`.
- `GET /api/book/{book_id}/chapters` — список глав + блокировки (учитывая прогресс пользователя). `user_id` берётся из заголовка `X-Telegram-User-Id`, но с валидацией: сервер требует `X-Telegram-Init-Data` в заголовках и верифицирует на каждый API-запрос (через зависимость `Depends(verify_user)`).
- `GET /api/chapter/{book_id}/{chapter_num}` — вопросы главы (без `keywords` и `reference_answer` — только `text` и `id`).
- `POST /api/answer` — `{book_id, chapter_num, question_id, answer_text}` → возвращает `{score, feedback}`. Сохраняет попытку в `attempts`.
- `POST /api/complete_chapter` — `{book_id, chapter_num, scores: [score1, score2, ...]}`. Сервер проверяет: `all(s == 2 for s in scores)`. Если да — отмечает главу как пройденную (UPDATE `chapter_progress` → `completed=1`). Если нет — не меняет прогресс. Возвращает `{passed: bool, total_percent: float, next_chapter_unlocked: bool}`.

Все API-роуты кроме `/` обязаны валидировать initData.

## 3. Авторизация (`backend/auth.py`)

Функция `verify_init_data(init_data_str: str, bot_token: str) -> dict | None`:
- Парсит query-string `init_data_str`
- Отделяет поле `hash`
- Сортирует оставшиеся ключи, собирает в строку `k=v\nk=v...`
- Секретный ключ: `hmac.new(b"WebAppData", bot_token.encode(), hashlib.sha256).digest()`
- Вычисляет HMAC-SHA256 этой строки с секретным ключом
- Сравнивает с `hash`. Если совпал — вернуть распарсенный `user` (JSON), иначе None.
- Проверить `auth_date` — не старше 24 часов.

FastAPI-зависимость `verify_user(request) -> dict`: берёт заголовок `X-Telegram-Init-Data`, прогоняет через `verify_init_data`, при провале → `HTTPException(401)`.

## 4. Оценка (`backend/grader.py`)

Перенести `bot/grader.py` как есть — функция `grade_answer(question, user_answer, use_ai=False, api_key=None)` уже работает. AI-режим оставить как опцию в `.env`, **по умолчанию выкл**.

## 5. Хранилище (`backend/storage.py`)

Перенести `bot/storage.py`. Одно изменение — в `upsert_chapter_progress`: поле `completed=1` выставляется **ТОЛЬКО если передать явный флаг `all_perfect=True`**. Сигнатура:
```python
async def mark_chapter_passed(user_id, book_id, chapter_num) -> None
```
Эта функция просто делает UPSERT в `chapter_progress` с `completed=1`, `best_avg_score=2.0`, `completed_at=datetime('now')`.

Никакой логики «частичного прохождения» — либо пройдено полностью, либо нет.

Добавить функцию `async def get_total_progress(user_id, book_id, total_chapters) -> float`:
возвращает `(completed_count / total_chapters) * 100`, округлённое до 1 знака.

## 6. Точка входа (`backend/run.py`)

Одним процессом запустить и бота, и FastAPI:
```python
async def main():
    await init_db()
    bot_task = asyncio.create_task(start_bot())    # polling
    server = uvicorn.Server(uvicorn.Config(app, host="0.0.0.0", port=8000, log_level="info"))
    await asyncio.gather(bot_task, server.serve())
```

## 7. Фронт (`webapp/index.html`, `app.js`, `style.css`)

### Экраны (SPA через div-переключение, без роутера)

1. **Loading** — пока идёт `/api/init`.
2. **Home** — большое SVG-кольцо с процентом (CSS `stroke-dasharray` анимация), под кольцом — заголовок книги, ниже список глав:
   - ✅ Пройдена — зелёный фон
   - 🔓 Доступна — обычный фон, кликабельна
   - 🔒 Закрыта — приглушённая, неактивна
3. **Chapter** — прогресс вопроса (`Вопрос 2/7`), текст вопроса, `<textarea>`, кнопка **«Ответить»**.
4. **Chapter Result**:
   - Если ВСЕ 2 балла → «🎉 Глава пройдена!», процент обновился (+анимация), кнопка «Следующая глава».
   - Если хоть один не 2 → «📖 Нужно перечитать главу — один из ответов был не точным», кнопка «Перечитать главу» (возвращает на **Home**, глава остаётся открытой для повтора), прогресс НЕ меняется.

### Стилизация

- Цвета — через переменные Telegram-темы:
  ```css
  background: var(--tg-theme-bg-color);
  color: var(--tg-theme-text-color);
  button: var(--tg-theme-button-color);
  ```
- Адаптация под мобильный (ширина 100%, max-width 480px).

### API-клиент (в `app.js`)

Все fetch-запросы шлют заголовок:
```js
headers: { 'X-Telegram-Init-Data': window.Telegram.WebApp.initData }
```

### Поведение

- `window.Telegram.WebApp.ready()` при загрузке
- `window.Telegram.WebApp.expand()` — развернуть во весь экран
- `window.Telegram.WebApp.BackButton.show()` на экране главы → клик возвращает на Home

## 8. Контент (`data/chapters.json`)

Наполнить **настоящими** 10 главами с 7-10 вопросами каждая по теме книги Эрика Райнерта «Почему бедные страны беднеют, а богатые страны богатеют». Для каждого вопроса:
- `text` — сам вопрос (открытый, не тест)
- `reference_answer` — эталонный ответ 2-3 предложения
- `keywords` — 4-7 ключевых слов/фраз
- `required_count` — сколько keywords должно сойтись для 2 баллов (обычно 3)

Если какие-то главы не знаешь точно — поставь заглушки `"title": "Глава N — уточни после прочтения"` с `"questions": []`. Заполнить в deep качестве хотя бы первые 3 главы.

## 10. Запуск (обнови `УСТАНОВКА.md`)

Пошагово:
1. Установить Python 3.11+ и ngrok (https://ngrok.com/download) + зарегистрироваться (бесплатно).
2. В PowerShell: `ngrok config add-authtoken <твой токен>` (одноразово).
3. `setup.bat` — ставит зависимости.
4. В отдельном терминале: `ngrok http 8000` — копируй HTTPS URL (например `https://abcd.ngrok-free.app`).
5. Вписать этот URL в `backend/.env` → `WEBAPP_URL=...`
6. Установить кнопку приложения в @BotFather:
   - `/mybots` → выбрать бот → **Bot Settings** → **Menu Button** → **Configure Menu Button** → прислать URL ngrok
7. `run.bat` — запустит бота + FastAPI.
8. Открыть бота в Telegram, нажать кнопку Menu слева — откроется приложение.

# КРИТЕРИИ ГОТОВНОСТИ

- `setup.bat` выполняется без ошибок.
- `run.bat` поднимает и бота, и сервер.
- Открытие WebApp в Telegram показывает экран Home с кольцом прогресса (0% на первом запуске).
- Первая глава доступна, вторая закрыта.
- Прохождение главы с ВСЕМИ 2-балльными ответами → +1 глава в счётчик, кольцо обновляется.
- Прохождение главы с хотя бы одним ≤1 баллом → прогресс НЕ меняется, открывается экран «нужно перечитать».
- Никакие API-роуты не отвечают без валидного initData.
- `bot/.env` остаётся вне git (проверь `.gitignore`).
