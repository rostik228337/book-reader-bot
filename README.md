# ПРОЕКТ КНИГА

Telegram Mini App для активного усвоения книги **«Почему бедные страны беднеют, а богатые страны богатеют»** (Эрик Райнерт) — через открытые вопросы после каждой главы.

## Как это работает

1. Открываешь бота в Telegram → жмёшь кнопку **📖 Читать** внизу чата
2. Выбираешь главу из списка
3. Бот задаёт 3–7 открытых вопросов по прочитанной главе
4. Отвечаешь своими словами в текстовом поле
5. Алгоритм проверяет ответ по ключевым словам (с учётом падежей — стемминг русского на JS)
6. Оценка `0 / 1 / 2` за каждый вопрос
7. Все ответы на `2` → глава закрыта, открывается следующая
8. Иначе → перечитай главу и попробуй снова

Прогресс сохраняется в **Telegram CloudStorage** — синхронится между твоими устройствами автоматически, без сервера.

## Архитектура

**Чистый статический сайт. Никакого бэкенда.**

Mini App — это просто HTML + CSS + JS, который раздаёт GitHub Pages. Telegram открывает его в своём WebView при нажатии кнопки меню. Вся логика (загрузка контента, оценка ответов, хранение прогресса) работает в браузере.

```
GitHub Pages (docs/)          Telegram
┌──────────────────┐         ┌─────────┐
│ index.html       │  ◀──── │ WebView  │
│ app.js           │         │          │
│ style.css        │         │ Menu btn │
│ chapters.json    │         │ "Читать" │
└──────────────────┘         └─────────┘
                                  │
                                  ▼
                           CloudStorage
                           (прогресс юзера)
```

## Контент (chapters.json)

Формат одного вопроса:

```json
{
  "id": 1,
  "text": "Вопрос пользователю",
  "reference_answer": "Эталонный развёрнутый ответ",
  "keywords": ["ключевое слово", "фраза", "термин"],
  "required_count": 3
}
```

**Как работает оценка:**
- Ответ юзера разбивается на слова → стемминг (отрезаются окончания, чтобы «возрастающая», «возрастающего», «возрастающие» свелись к одному корню)
- Каждое ключевое слово из `keywords` проверяется на вхождение в ответ (все корни ключевой фразы должны найтись)
- Если найдено `>= required_count` → **2 балла**
- Если найдено `required_count - 1` → **1 балл**
- Иначе → **0 баллов**

Для прохождения главы нужно **2 балла за каждый вопрос**. Иначе «перечитай главу».

## Обновление контента

Правишь `docs/chapters.json` → `git push` → через ~30 сек GitHub Pages пересобирается → Mini App подхватывает свежие вопросы при следующем открытии.

## Сброс прогресса

Через DevTools (Telegram Desktop: правый клик в Mini App → Inspect):
```js
await window.Telegram.WebApp.CloudStorage.removeItem("bookreader_progress_v1");
```

Или удалить и перезайти в чат с ботом.
