# Техническое задание

## 1. Суть проекта
Telegram-бот, который помогает пользователю глубже усваивать книги через открытые вопросы после каждой главы. Пользователь пишет ответ своими словами — бот оценивает и либо пропускает дальше, либо просит перечитать главу.

Первая книга в базе: **«Почему бедные страны беднеют, а богатые страны богатеют»** — Эрик Райнерт.

## 2. Цель
Не развлечение, не тесты с вариантами. Инструмент для активного запоминания и понимания. Рассчитан на одного пользователя (лично себя), но с возможностью добавлять новые книги и пользователей в будущем.

## 3. Формат
**Telegram-бот** — выбран как самый быстрый и простой MVP. Не требует регистрации, не требует хостинга сайта, работает на любом устройстве.

## 4. Функционал (MVP)

### 4.1. Навигация
- `/start` — приветствие, список доступных книг
- Выбор книги → список глав
- Выбор главы → старт теста

### 4.2. Прохождение главы
- Бот задаёт вопросы один за другим (3–7 вопросов на главу)
- На каждый вопрос пользователь пишет ответ текстом
- После всех вопросов — итоговая оценка по главе

### 4.3. Оценка ответа
**Шкала 0/1/2** для каждого вопроса:
- `0` — не ответил / полностью мимо
- `1` — частично понял, но не хватает ключевых смыслов
- `2` — нормально, ключевые мысли уловлены

**Как оценивается (режимы переключаются в настройках):**

**Режим А — без ИИ (по умолчанию, бесплатный)**
- В `chapters.json` для каждого вопроса задано:
  - `keywords` — список ключевых слов / фраз
  - `required_count` — сколько должно быть найдено для зачёта
- Алгоритм:
  - Привести ответ пользователя к нижнему регистру + лемматизация (pymorphy3 для русского)
  - Посчитать совпадения с `keywords` (тоже лемматизированными)
  - `найдено >= required_count` → 2 балла
  - `найдено == required_count - 1` → 1 балл
  - меньше → 0 баллов

**Режим Б — с ИИ (опционально, включается через `.env`)**
- Использовать OpenAI API (gpt-4o-mini — самый дешёвый)
- В промт передаётся: вопрос, эталонный ответ (из `chapters.json`), ответ пользователя
- Модель возвращает JSON: `{"score": 0|1|2, "feedback": "..."}`
- Если API недоступен / ошибка → откат на режим А

### 4.4. Логика прогресса
- Средний балл по главе:
  - `>= 1.5` → глава пройдена, открывается следующая
  - `< 1.5` → бот пишет «Перечитай главу и попробуй снова»
- Глава всегда доступна для повторного прохождения
- Следующая глава не открыта, пока не пройдена предыдущая

### 4.5. Минимальный UI (Telegram)
- **Главный экран:** список книг (инлайн-кнопки)
- **Экран книги:** список глав (✅ пройдена / 🔓 доступна / 🔒 закрыта)
- **Экран главы:** вопросы по одному, бот ждёт текст ответа
- **Итог:** оценка + фидбек + кнопка «Следующая глава» или «Перечитать»

## 5. Данные

### 5.1. Контент (`data/chapters.json`)
Статический файл, редактируется вручную. Структура:
```json
{
  "books": [
    {
      "id": "reinert_poor_rich",
      "title": "Почему бедные страны беднеют...",
      "author": "Эрик Райнерт",
      "chapters": [
        {
          "number": 1,
          "title": "Название главы",
          "questions": [
            {
              "id": 1,
              "text": "Вопрос?",
              "reference_answer": "Эталонный развёрнутый ответ (для режима ИИ)",
              "keywords": ["слово1", "фраза два", "ключевой смысл"],
              "required_count": 2
            }
          ]
        }
      ]
    }
  ]
}
```

### 5.2. Прогресс пользователя (`data/progress.db`, SQLite)
Таблицы:
- `users(user_id, username, created_at, ai_mode_enabled)`
- `attempts(id, user_id, book_id, chapter_num, question_id, answer_text, score, feedback, created_at)`
- `chapter_progress(user_id, book_id, chapter_num, best_avg_score, completed, completed_at)`

## 7. Расширения (не в MVP, на будущее)
- Статистика прогресса (команда `/stats`)
- Возврат к сложным вопросам через неделю (интервальное повторение)
- Добавление книг через админ-команду
- Экспорт всех ответов в файл

## 8. Приоритеты
1. **Просто.** Один файл main.py с хендлерами, минимум абстракций.
2. **Быстро запустить.** От «установил Python» до «бот работает» — 20 минут.
3. **Работает для одного пользователя.** Не нужна нагрузка, аналитика, админка.
4. **Легко добавить ИИ.** Оценка инкапсулирована в одну функцию `grade_answer()` — достаточно переключить флаг.
