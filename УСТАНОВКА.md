# Установка — через GitHub Pages (без Python, без ngrok)

Один раз настроил → потом только `git push` для обновлений.

## ШАГ 1. Получить токен Telegram-бота

1. Открой Telegram → найди **@BotFather**
2. `/newbot` → придумай имя и username (должен кончаться на `bot`)
3. Сохрани токен вида `123456789:AAH...`

## ШАГ 2. Залить проект в GitHub

Если ещё нет аккаунта — зарегистрируйся на https://github.com.

В консоли (PowerShell) в папке проекта:

```powershell
cd "C:\Users\User\Documents\Claude\Projects\ПРОЕКТ КНИГА"
git init
git add .
git commit -m "init"
git branch -M main
git remote add origin https://github.com/<твой-ник>/<название-репо>.git
git push -u origin main
```

Если просит логин — используй **Personal Access Token** вместо пароля: GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new.

## ШАГ 3. Включить GitHub Pages

1. Открой свой репозиторий на github.com
2. Settings → Pages
3. **Source:** Deploy from a branch
4. **Branch:** `main` / **Folder:** `/docs`
5. Save

Через 30–60 секунд вверху страницы появится зелёная плашка с URL:
```
https://<твой-ник>.github.io/<название-репо>/
```

**Проверь:** открой этот URL в браузере — должна показаться заглушка «Открой через Telegram» или UI книги (в зависимости от поведения Telegram SDK в обычном браузере).

## ШАГ 4. Прикрутить Mini App к боту

В BotFather:
1. `/mybots` → выбери своего бота
2. **Bot Settings** → **Menu Button** → **Configure menu button**
3. URL: вставь ссылку с шага 3
4. Название: `📖 Открыть книгу` (или что нравится)

## ШАГ 5. Проверить

1. Открой бота в Telegram
2. Внизу появилась кнопка меню → жми
3. Откроется Mini App с кнопкой «Главы»

## Обновление контента / главы

Отредактировал `docs/chapters.json` локально:

```powershell
cd "C:\Users\User\Documents\Claude\Projects\ПРОЕКТ КНИГА"
git add docs/chapters.json
git commit -m "update chapter N"
git push
```

Через ~30 секунд GitHub Pages пересоберёт — в Mini App новый контент подтянется при следующем открытии (может понадобиться смахнуть Mini App и открыть заново, чтобы сбросить кэш).

## Локальный превью (без Telegram)

```powershell
cd "C:\Users\User\Documents\Claude\Projects\ПРОЕКТ КНИГА\docs"
python -m http.server 8000
```

Открой `http://localhost:8000` — UI увидишь, но функции Telegram (CloudStorage, BackButton) работать не будут. Прогресс временно уедет в localStorage браузера.

## Что где хранится

| Что | Где | Зачем |
|---|---|---|
| Главы и вопросы | `docs/chapters.json` | Контент. Правишь вручную, коммитишь. |
| Прогресс | Telegram CloudStorage у каждого юзера | Сохраняется между устройствами, живёт у Telegram. |
| Bot token | В BotFather | Mini App сам НЕ знает токен — всё клиентское, ему токен не нужен. |

## Частые проблемы

**Pages не поднимается / 404** — проверь, что включил `/docs`, а не `/(root)`. Обновления через 30–60 сек.

**Mini App показывает белый экран** — открой URL в обычном браузере, посмотри консоль (F12). Чаще всего это путь к `chapters.json` (должен грузиться через относительный путь).

**Прогресс сбрасывается** — если открываешь Mini App не через Telegram, CloudStorage недоступен и идёт фоллбэк на localStorage, который привязан к браузеру.
